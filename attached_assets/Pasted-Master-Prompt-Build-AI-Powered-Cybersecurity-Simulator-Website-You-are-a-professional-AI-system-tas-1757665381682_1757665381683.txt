Frontend: Tailwind CSS + React or TypeScript

Database: SQL (PostgreSQL preferred)

Connection: Frontend must interact with backend and database using REST APIs

Containerization: Docker Compose for orchestration

WAF: Nginx with ModSecurity + OWASP CRS

IDS: Suricata with eve.json output

SIEM: Lightweight ELK stack or Wazuh/Filebeat

Red Team Tools: Container with sqlmap, hydra, and attack scripts

Blue Team Scripts:

block_ip.sh

isolate_container.sh

collect_logs.sh

Detection Rules: SQLi, brute-force, suspicious uploads, phishing URLs

Malware Scanning: ClamAV + YARA (optional)

Phishing Detection: Heuristics + optional VirusTotal integration

📄 Output Requirements
Provide exact copy-paste files: docker-compose.yml, backend code, frontend code, SQL schema, config files

Include step-by-step commands to deploy and run

Explain each file in one short paragraph

Simulate RCE safely (e.g., wget to test server)

No destructive payloads

All components must be open-source and memory-light

Act as a cybersecurity architect and full-stack developer

Assume user has basic Linux and Docker knowledge

Prioritize clarity, reproducibility, and safety

Do not include external setup steps or vague instructions

🎯 Goal
Deliver a fully working prototype that:

Detects threats
